📌 Module 02

Text Preprocessing

Before feeding text into any model, we need to clean and standardize it. Garbage in = garbage out — this step is critical!

🌍 Real World Example

Imagine training a spam detector on emails. Without preprocessing, "FREE!!!" and "free" are treated as different words — but they mean the same thing. Preprocessing fixes this.

Tokenization splits text into individual units (tokens). Think of it like cutting a sentence into individual LEGO bricks before building something.

PYTHON

Stop words are common words (like "is", "the", "a") that carry little meaning. Removing them reduces noise and speeds up processing.

PYTHON
from nltk.corpus import stopwords from nltk.tokenize import word_tokenize import re nltk.download('stopwords') text = "The cat sat on the mat and it was very comfortable" # Get English stop words stop_words = set(stopwords.words('english')) print(f"Stop words sample: {list(stop_words)[:10]}") # Clean and remove stop words text_clean = re.sub(r'[^a-zA-Z\s]', '', text.lower()) tokens = word_tokenize(text_clean) filtered = [w for w in tokens if w not in stop_words] print(f"Before: {tokens}") # ['the', 'cat', 'sat', 'on', 'the', 'mat', 'and', 'it', 'was', 'very', 'comfortable'] print(f"After: {filtered}") # ['cat', 'sat', 'mat', 'comfortable'] ← much cleaner!

Stemming chops words to their root (sometimes rough). Lemmatization uses grammar rules to find the true base form. Lemmatization is slower but more accurate.

PYTHON
from nltk.stem import PorterStemmer, SnowballStemmer from nltk.stem import WordNetLemmatizer ps = PorterStemmer() lemmatizer = WordNetLemmatizer() words = ["running", "studies", "better", "flies", "caring"] print(f"{'Word':<12} {'Stem':<12} {'Lemma'}") print("-" * 36) for word in words: stem = ps.stem(word) lemma = lemmatizer.lemmatize(word, pos='v') print(f"{word:<12} {stem:<12} {lemma}") # Word Stem Lemma # ------------------------------------ # running run run # studies studi study ← stem is messy! # better better be ← lemma is smarter # flies fli fly # caring care care

Part-of-Speech (POS) tagging labels each word as noun, verb, adjective, etc. This helps with understanding sentence structure and named entity recognition.

📌 Module 03

TF-IDF

Term Frequency–Inverse Document Frequency is a way to measure how important a word is in a document relative to a collection of documents. It's the backbone of search engines!

🌍 Real World Example

When you Google "python programming", Google uses TF-IDF (and more) to rank which pages are most relevant. Words like "python" that appear frequently in a document but rarely across all pages get a high TF-IDF score — making it relevant.

The Math Behind TF-IDF

Step 1: Term Frequency (TF)
TF(t, d) = count(t in d) / total_words(d)
tterm (word)
ddocument
count(t in d)times word appears
total_words(d)total words in doc
Step 2: Inverse Document Frequency (IDF)
IDF(t, D) = log( N / df(t) )
Ntotal documents
df(t)docs containing term t
log()dampens large values
Final Score
TF-IDF(t, d, D) = TF(t, d) × IDF(t, D)

High score = word is important in THIS doc but rare across all docs

PYTHON
from sklearn.feature_extraction.text import TfidfVectorizer import pandas as pd import numpy as np # Example corpus (collection of documents) corpus = [ "the cat sat on the mat", "the dog sat on the log", "cats and dogs are great pets", "the cat chased the dog around" ] # Create TF-IDF matrix vectorizer = TfidfVectorizer( stop_words='english', # remove common words max_features=10, # keep top 10 words ngram_range=(1, 2), # unigrams + bigrams smooth_idf=True # avoid division by zero ) tfidf_matrix = vectorizer.fit_transform(corpus) feature_names = vectorizer.get_feature_names_out() # Display as DataFrame df = pd.DataFrame( tfidf_matrix.toarray().round(3), columns=feature_names, index=[f"Doc {i+1}" for i in range(len(corpus))] ) print(df.to_string()) # Get top words for document 1 doc_idx = 0 scores = dict(zip(feature_names, tfidf_matrix[doc_idx].toarray()[0])) top = sorted(scores.items(), key=lambda x: x[1], reverse=True) print(f"Top words in Doc 1: {top[:3]}") # Top words in Doc 1: [('mat', 0.635), ('cat', 0.523), ('sat', 0.412)]

📌 Module 04

Word Embeddings

Instead of counting words, embeddings represent each word as a dense vector of numbers — capturing semantic meaning. Similar words end up close together in vector space.

🌍 Real World Example

In a word embedding model: king - man + woman ≈ queen. That's the magic — these vectors capture relationships and analogies that simple counting can't. Spotify uses embeddings to understand song similarity for recommendations!

One-Hot Encoding (old way)

cat → [1, 0, 0, 0, 0, ...] (sparse) dog → [0, 1, 0, 0, 0, ...] kitty → [0, 0, 1, 0, 0, ...] # "cat" and "kitty" look completely # different — loses all meaning!

Word Embeddings (new way)

cat → [0.23, -0.51, 0.89, 0.12] dog → [0.21, -0.48, 0.87, 0.08] kitty → [0.24, -0.53, 0.91, 0.11] # cat and kitty are CLOSE in space! # Similarity preserved ✓

Word2Vec — How It Works

Word2Vec trains a neural network to predict surrounding words (CBOW) or predict a word from its context (Skip-gram). The hidden layer weights become the embeddings.

Cosine Similarity — Measuring Word Closeness

Cosine Similarity Formula
sim(A, B) = (A · B) / (||A|| × ||B||)
A · Bdot product
||A||magnitude of A
range: [-1, 1]1 = identical
PYTHON
from gensim.models import Word2Vec from sklearn.metrics.pairwise import cosine_similarity import numpy as np # Train Word2Vec on sentences sentences = [ ["the", "cat", "sat", "on", "the", "mat"], ["the", "king", "is", "a", "man"], ["the", "queen", "is", "a", "woman"], ["cats", "and", "dogs", "are", "pets"], ] model = Word2Vec( sentences, vector_size=100, # dimension of embedding window=5, # context window size min_count=1, # minimum word frequency workers=4, # parallel training sg=1 # Skip-gram (1) or CBOW (0) ) # Find similar words similar = model.wv.most_similar('cat', topn=3) print(f"Words similar to 'cat': {similar}") # Word analogies: king - man + woman = ? result = model.wv.most_similar( positive=['king', 'woman'], negative=['man'], topn=1 ) print(f"king - man + woman = {result[0][0]}") # king - man + woman = queen 🎉 # Cosine similarity between two words sim = model.wv.similarity('cat', 'dog') print(f"Similarity(cat, dog) = {sim:.3f}") # ~0.85 # Use pre-trained GloVe/Word2Vec (practical) import gensim.downloader pretrained = gensim.downloader.load('word2vec-google-news-300') print(pretrained.most_similar('python', topn=5))

📌 Module 05

Text Classification

Text classification assigns a category to a piece of text. It powers spam detection, content moderation, customer support routing, and much more.

🌍 Real World Example

Gmail's spam filter is a text classifier. It reads your email content, checks patterns it has learned from millions of labeled spam/not-spam emails, and assigns a label. That's exactly what we're about to build!

3 Approaches — Simple to Powerful

💡 Intuition

Naive Bayes asks: "Given this word, which class is more probable?" It uses Bayes' theorem and assumes all words are independent (the "naive" part). Despite this simplification, it works surprisingly well for text!

Bayes' Theorem for Classification
P(class | text) ∝ P(class) × ∏ P(word | class)
PYTHON
from sklearn.naive_bayes import MultinomialNB from sklearn.feature_extraction.text import TfidfVectorizer from sklearn.model_selection import train_test_split from sklearn.metrics import classification_report # Dataset: email spam detection texts = [ "Win FREE money now! Click here!", "Hey can we meet tomorrow for lunch?", "URGENT: Claim your lottery prize", "Please review the attached report", "Buy cheap medications online now", "Team meeting at 3pm in room 201", "You've been selected! Cash waiting", "Can you send me the project files?", ] labels = ["spam","ham","spam","ham","spam","ham","spam","ham"] # Split data X_train, X_test, y_train, y_test = train_test_split( texts, labels, test_size=0.25, random_state=42 ) # Vectorize text vectorizer = TfidfVectorizer() X_train_vec = vectorizer.fit_transform(X_train) X_test_vec = vectorizer.transform(X_test) # Train Naive Bayes clf = MultinomialNB(alpha=1.0) # alpha = Laplace smoothing clf.fit(X_train_vec, y_train) # Predict & evaluate y_pred = clf.predict(X_test_vec) print(classification_report(y_test, y_pred)) # Predict new text new_text = ["CONGRATULATIONS you won a prize!"] pred = clf.predict(vectorizer.transform(new_text)) print(f"Prediction: {pred[0]}") # → spam
💡 Intuition

Logistic Regression learns a weight for each word feature. It's like saying "the word 'free' contributes +2.5 toward SPAM, while 'meeting' contributes -1.8". These weights are learned from training data.

Sigmoid Function (maps to 0-1 probability)
σ(z) = 1 / (1 + e^(-z)) where z = w·x + b
PYTHON
from sklearn.linear_model import LogisticRegression from sklearn.feature_extraction.text import TfidfVectorizer import numpy as np # After vectorizing (same as above), train LogReg clf = LogisticRegression( C=1.0, # regularization (smaller = stronger) max_iter=1000, # iterations for optimizer solver='lbfgs' # optimization algorithm ) clf.fit(X_train_vec, y_train) # Get probability estimates (more useful than just labels) probs = clf.predict_proba(X_test_vec) for text, prob in zip(X_test, probs): print(f"'{text[:30]}...' → spam: {prob[1]:.1%}") # Inspect most influential words feature_names = vectorizer.get_feature_names_out() coefs = clf.coef_[0] top_spam = np.argsort(coefs)[-5:] print("Top SPAM words:", feature_names[top_spam]) # ['free', 'win', 'prize', 'click', 'cash']

A proper ML pipeline chains preprocessing, vectorization, and classification into a single reusable object. This prevents data leakage and makes deployment clean.

PYTHON — Full Production Pipeline
from sklearn.pipeline import Pipeline from sklearn.feature_extraction.text import TfidfVectorizer from sklearn.linear_model import LogisticRegression from sklearn.model_selection import cross_val_score, GridSearchCV from sklearn.metrics import classification_report, confusion_matrix import joblib # ────────────────────────────────────────── # 1. Build pipeline # ────────────────────────────────────────── pipeline = Pipeline([ ('tfidf', TfidfVectorizer( stop_words='english', ngram_range=(1, 2), max_features=50000 )), ('clf', LogisticRegression(max_iter=1000)) ]) # ────────────────────────────────────────── # 2. Hyperparameter tuning with GridSearch # ────────────────────────────────────────── param_grid = { 'tfidf__max_features': [10000, 50000], 'tfidf__ngram_range': [(1,1), (1,2)], 'clf__C': [0.1, 1.0, 10.0], } grid_search = GridSearchCV( pipeline, param_grid, cv=5, scoring='f1_macro' ) grid_search.fit(X_train, y_train) print(f"Best params: {grid_search.best_params_}") # ────────────────────────────────────────── # 3. Evaluate # ────────────────────────────────────────── best = grid_search.best_estimator_ y_pred = best.predict(X_test) print(classification_report(y_test, y_pred)) # ────────────────────────────────────────── # 4. Save and load model (production!) # ────────────────────────────────────────── joblib.dump(best, 'spam_classifier.pkl') loaded_model = joblib.load('spam_classifier.pkl') print(loaded_model.predict(["Congratulations! You won a prize"]))

📌 Module 07

Attention Mechanism

Attention allows the model to "look at" all other words when processing a word. It learns WHICH words to focus on — this is the core innovation of Transformers.

🌍 Real World Analogy

When you read "The animal didn't cross the street because it was too tired", you know "it" refers to "animal". Attention lets the model make this connection automatically!

The Math: Query, Key, Value

Scaled Dot-Product Attention
Attention(Q, K, V) = softmax( Q·Kᵀ / √d_k ) · V
Q (Query)"What am I looking for?"
K (Key)"What do I have?"
V (Value)"What do I return?"
√d_kscaling factor

PYTHON — Attention from Scratch
import numpy as np def softmax(x): e_x = np.exp(x - np.max(x)) return e_x / e_x.sum() def scaled_dot_product_attention(Q, K, V): d_k = Q.shape[-1] # Compute attention scores scores = np.matmul(Q, K.T) / np.sqrt(d_k) # Convert to probabilities weights = np.array([softmax(row) for row in scores]) # Weighted sum of values output = np.matmul(weights, V) return output, weights # Example: 4 tokens, 8-dimensional embeddings seq_len, d_model, d_k = 4, 8, 8 np.random.seed(42) X = np.random.randn(seq_len, d_model) # Weight matrices (learned during training) W_Q = np.random.randn(d_model, d_k) W_K = np.random.randn(d_model, d_k) W_V = np.random.randn(d_model, d_k) # Compute Q, K, V projections Q = X @ W_Q K = X @ W_K V = X @ W_V output, attn_weights = scaled_dot_product_attention(Q, K, V) print(f"Attention weights:\n{attn_weights.round(3)}") print(f"Output shape: {output.shape}")
📌 Module 08

BERT & HuggingFace in Practice

BERT (Bidirectional Encoder Representations from Transformers) is pre-trained on 3.3 billion words. You can fine-tune it for your specific task in minutes — not months!

🌍 Real World Example

Google uses BERT in its search engine to understand search queries better. A query like "can you get medicine for someone pharmacy" — BERT understands "for someone" is the key context, not just keyword matching.

BERT Fine-Tuning Pipeline

STEP 1
Pre-trained BERT
Load model trained on billions of words. All language knowledge is already there.
STEP 2
Tokenize Data
Use BERT's special tokenizer (WordPiece). Handles [CLS] and [SEP] tokens.
STEP 3
Add Classifier
Add a linear layer on top of [CLS] token representation.
STEP 4
Fine-tune
Train on your labeled data for 2-4 epochs with low learning rate.
STEP 5
Deploy!
Export model, build inference endpoint. Serve predictions.
PYTHON — BERT Sentiment Classifier (HuggingFace)
from transformers import ( BertTokenizer, BertForSequenceClassification, Trainer, TrainingArguments ) from datasets import load_dataset import torch # ────────────────────────────────────────── # 1. Load Pre-trained BERT # ────────────────────────────────────────── MODEL_NAME = "bert-base-uncased" tokenizer = BertTokenizer.from_pretrained(MODEL_NAME) model = BertForSequenceClassification.from_pretrained( MODEL_NAME, num_labels=2 # positive / negative ) # ────────────────────────────────────────── # 2. Load & Tokenize Dataset # ────────────────────────────────────────── dataset = load_dataset("imdb") # 50k movie reviews def tokenize_fn(batch): return tokenizer( batch["text"], truncation=True, padding="max_length", max_length=512 ) tokenized = dataset.map(tokenize_fn, batched=True) # ────────────────────────────────────────── # 3. Training Arguments # ────────────────────────────────────────── training_args = TrainingArguments( output_dir="./results", num_train_epochs=3, per_device_train_batch_size=16, per_device_eval_batch_size=64, learning_rate=2e-5, # small LR for fine-tuning! weight_decay=0.01, evaluation_strategy="epoch", save_strategy="epoch", load_best_model_at_end=True, report_to="none" ) # ────────────────────────────────────────── # 4. Train! # ────────────────────────────────────────── trainer = Trainer( model=model, args=training_args, train_dataset=tokenized["train"], eval_dataset=tokenized["test"], ) trainer.train() # ~93% accuracy on IMDB! # ────────────────────────────────────────── # 5. Quick inference with pipeline API # ────────────────────────────────────────── from transformers import pipeline sentiment = pipeline( "sentiment-analysis", model="distilbert-base-uncased-finetuned-sst-2-english" ) results = sentiment([ "This movie was absolutely fantastic!", "What a waste of two hours.", "It was okay, not great not bad." ]) for r in results: print(f"{r['label']} ({r['score']:.1%})") # POSITIVE (99.8%) # NEGATIVE (99.9%) # NEGATIVE (59.2%)

🎯 Quick Reference Cheat Sheet

When to use TF-IDF + Traditional ML

  • Small dataset (<10k samples)
  • Fast training needed
  • Interpretability required
  • Low compute budget

When to use Transformers (BERT)

  • Large dataset available
  • Context matters (ambiguous words)
  • State-of-the-art accuracy needed
  • Complex tasks (QA, NER, translation)
