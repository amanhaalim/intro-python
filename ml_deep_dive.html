Topic 01 / 07 — Supervised Learning

Regression Models

Predicting a continuous number from input features — think house prices, temperatures, and scores.

Plain English

What is Regression?

Real-world analogy

You want to predict a house price. You consider its size (1200 sq ft), number of bedrooms (3), and location. Regression learns the pattern from thousands of past sales and draws the best-fit line so it can estimate any new house's price.

Other examples: predicting tomorrow's temperature, a student's exam score, a car's fuel efficiency, or estimated delivery time. The key idea: the output is a number on a continuous scale — not a category.

Technical Deep Dive

Linear Regression

Finds coefficients (β values) that define the best straight line through data points, minimizing prediction error.

prediction equation ŷ = β₀ + β₁x₁ + β₂x₂ + ... + βₙxₙ

β₀ = intercept (base value when all features = 0)  ·  β₁…βₙ = how much each feature contributes per unit

loss function — Mean Squared Error MSE = (1/n) × Σᵢ ( yᵢ − ŷᵢ )²

We minimize MSE using gradient descent — nudging β values step by step in the direction that reduces error, like rolling a ball downhill to its lowest point.

goodness of fit R² = 1 − ( SS_residual / SS_total )    range: 0 → 1   (1 = perfect fit)
gradient descent update rule βⱼ ← βⱼ − α × ∂MSE/∂βⱼ    where α = learning rate

Types of Regression

Linear

Straight-line relationship. Works when output scales proportionally with inputs.

ŷ = β₀ + β₁x

Polynomial

Curved relationship. Adds powers of x as new features to capture bends.

ŷ = β₀ + β₁x + β₂x²

Ridge (L2)

Prevents overfitting by penalizing large β values. Shrinks all coefficients toward zero.

Loss + λΣβ²

Lasso (L1)

Like Ridge but zeros out unimportant features entirely — automatic feature selection.

Loss + λΣ|β|
Topic 02 / 07 — Supervised Learning

Classification Models

Predicting which category or class a data point belongs to.

Plain English

What is Classification?

Real-world analogy

A doctor looks at your symptoms and says "flu" or "cold." An email filter decides "spam" or "not spam." A photo app identifies "cat," "dog," or "bird." Classification puts things into discrete boxes — not on a number line.

Binary classification = 2 classes (spam/not spam)  ·  Multi-class = 3+ classes (cat/dog/bird)  ·  Multi-label = multiple tags per sample

Technical Deep Dive

Logistic Regression — despite the name, it classifies!

Uses the sigmoid function to squeeze any real number into a probability between 0 and 1. If probability > 0.5, predict class 1.

sigmoid (logistic) function σ(z) = 1 / ( 1 + e−z )    where z = β₀ + β₁x₁ + ... + βₙxₙ
Binary Cross-Entropy loss L = −[ y · log(ŷ) + (1−y) · log(1−ŷ) ]

When y=1 (true positive), we penalize low confidence predictions. When y=0 (true negative), we penalize high confidence predictions. The model learns to be confident and correct simultaneously.

Decision rule if σ(z) > threshold (0.5) → predict class 1   else → predict class 0

Other Popular Classifiers

Decision Tree

Asks yes/no questions like a flowchart. "Age > 30?" → "Income > 50k?" → approve. Intuitive and explainable.

Random Forest

Hundreds of decision trees each vote. Majority wins. Much harder to overfit — wisdom of the crowd.

SVM (Support Vector Machine)

Finds the widest possible margin between classes. Kernels allow complex non-linear boundaries.

k-Nearest Neighbors

"You are like your k nearest neighbors." Lazy learner — memorizes training data, classifies at query time by majority vote.

Topic 03 / 07 — Unsupervised Learning

Clustering

Finding natural groups in data — without any predefined labels.

Plain English
Real-world analogy

A supermarket never told the computer who is a "budget shopper" vs a "premium buyer." Clustering analyzes purchase history and discovers these groups on its own — like sorting a pile of socks by color when no one told you what colors exist. Pure, unsupervised pattern-finding.

Unlike classification, this is unsupervised — no labels, no correct answers. The algorithm invents the categories.

Other real-world uses: customer segmentation, anomaly/fraud detection, gene expression grouping, news topic discovery, image compression (color quantization).

Technical — K-Means Algorithm

How K-Means works

1
Choose k (number of clusters). Place k centroids randomly in the feature space.
2
Assign each point to its nearest centroid using Euclidean distance.
3
Move each centroid to the mean position of all points assigned to it.
4
Repeat steps 2–3 until centroids stop moving (convergence).
Euclidean distance d(a, b) = √( (x₂−x₁)² + (y₂−y₁)² )
Objective — minimize WCSS (Within-Cluster Sum of Squares) J = Σᵢ₌₁ᵏ   Σx∈Cᵢ ‖x − μᵢ‖²

The Elbow Method: plot WCSS vs k. The "elbow" — where gains in WCSS diminish sharply — reveals the optimal k.

Other Clustering Methods

DBSCAN

Density-based. Discovers arbitrarily-shaped clusters. Automatically labels outliers as "noise." No need to specify k!

Hierarchical (Agglomerative)

Builds a dendrogram — a tree of cluster merges. Cut at any level to get your desired number of groups.

Topic 04 / 07 — Model Evaluation

Confusion Matrix

A detailed scorecard showing exactly where your classifier gets it right — and wrong.

Plain English
Real-world analogy

A COVID test can fail in two distinct ways: saying you're sick when you're healthy (false alarm), or saying you're healthy when you're actually sick (missed detection). A confusion matrix lays out all four possible outcomes so you know precisely what kind of errors the model makes — not just how often it's wrong.

Technical — The Four Cells
Predicted: PositivePredicted: Negative
Actual: Positive TPTrue Positive FNFalse Negative
Actual: Negative FPFalse Positive TNTrue Negative
TRUE POSITIVE (TP)

Predicted positive + Actually positive. A correct hit. "Diagnosed flu → Had flu."

TRUE NEGATIVE (TN)

Predicted negative + Actually negative. Correct rejection. "No flu → no flu."

FALSE POSITIVE (FP) — Type I Error

Predicted positive → Actually negative. False alarm. "Said flu → Was just a cold."

FALSE NEGATIVE (FN) — Type II Error

Predicted negative → Actually positive. Missed it. "Said healthy → Had cancer."

Topic 05 / 07 — Model Evaluation

Precision, Recall & F1 Score

Going beyond accuracy to understand the quality and completeness of your predictions.

Plain English — Why Accuracy Isn't Enough
The 99% accuracy trap

If 99% of emails are not spam, a model that labels everything "not spam" achieves 99% accuracy — yet catches zero spam! That's a useless model with deceptive accuracy. Precision and Recall reveal the full story.

PRECISION

Of all the cases I predicted positive, how many actually were? Measures quality of my alarms.

"When I cry wolf, am I actually right?"

RECALL (SENSITIVITY)

Of all actual positives, how many did I catch? Measures completeness — did I miss any?

"Do I catch all the wolves that exist?"

Mathematical Formulas
AccuracyAccuracy = (TP + TN) / (TP + TN + FP + FN)
Precision — purity of positive predictionsPrecision = TP / (TP + FP)     ← "of my positive calls, fraction that were right"
Recall — coverage of actual positivesRecall = TP / (TP + FN)     ← "of all real positives, fraction I found"
F1 Score — harmonic mean of P and RF1 = 2 × (Precision × Recall) / (Precision + Recall)

F1 uses the harmonic mean — it punishes extreme imbalances more than arithmetic mean. A model with P=1.0 and R=0.01 only scores F1=0.02. You need both to score well.

Fβ — weighted F scoreFβ = (1+β²) × P × R / (β²·P + R)    [β>1 weights recall; β<1 weights precision]

When to prioritize what

Maximize Precision

When false alarms are costly. Spam filters — you don't want real emails deleted. Legal or compliance flagging systems.

Maximize Recall

When missing a positive is dangerous. Cancer or fraud detection — missing a case is worse than a false alarm.

Balance with F1

When both errors matter equally. General NLP tasks, most business classifiers, benchmark evaluations.

AUC-ROC

Area under the ROC curve — model quality across all thresholds. 0.5 = random guessing, 1.0 = perfect. Threshold-independent.

Topic 06 / 07 — Model Evaluation

Cross-Validation

A robust technique to estimate how well a model truly generalizes to unseen data.

Plain English
Real-world analogy

You're studying for a board exam. If you only practice the same 10 questions over and over, you might memorize them without truly understanding the material. Cross-validation forces the model to be tested on data it has never seen — multiple times — giving you a much more honest performance estimate.

A simple train/test split is like taking one exam. Cross-validation is like taking 5 different exams on different question sets and averaging your score — far more reliable and trustworthy.

Technical — K-Fold Cross-Validation
1
Shuffle the full dataset and divide it into k equal-sized folds.
2
Train the model on k−1 folds. Evaluate on the remaining 1 fold. Record validation score.
3
Rotate: use a different fold as the test set. Repeat until every fold has been the test set once.
4
Final reported score = mean ± std deviation across all k scores.
CV mean scoreCV = (1/k) × Σᵢ₌₁ᵏ score(foldᵢ)    typical: k = 5 or 10
Interpreting resultsHigh variance (large std) across folds → model is unstable → likely overfitting
Consistently low score → model is too simple → likely underfitting

Variants of Cross-Validation

Stratified K-Fold

Ensures each fold preserves the class ratio of the full dataset. Essential for imbalanced classes.

Leave-One-Out (LOOCV)

k = n (sample count). Every single point is tested once. Very thorough but computationally expensive.

Time-Series CV

Always train on the past, test on future. Never "look ahead." Walk-forward / expanding window validation.

Nested CV

Outer loop evaluates the model; inner loop tunes hyperparameters. Prevents information leakage when tuning and evaluating.

Topic 07 / 07 — Optimization

Hyperparameter Tuning

Finding the model settings that unlock peak performance — the art of dialing in your ML system.

Plain English
Real-world analogy

When baking bread, you control oven temperature, flour amount, and kneading time. These are your "hyperparameters." The flour-to-yeast ratio once baking starts are "parameters" (learned in the process). Hyperparameter tuning = systematically testing different oven temperatures and times until you find the perfect loaf.

Model parameters are learned automatically from data (e.g., β weights in regression). Hyperparameters are set by you before training begins — they control how learning happens.

Three Search Strategies
1
Grid Search — Exhaustively tries every combination. Guaranteed to find the best in the specified grid. But: 5 values × 5 values × 5 values = 125 training runs. Expensive with many parameters.
2
Random Search — Samples combinations randomly. Bergstra & Bengio (2012) proved random search often finds near-optimal results with far fewer trials. Works especially well when only a few hyperparameters truly matter.
3
Bayesian Optimization — Builds a probabilistic surrogate model (often a Gaussian Process) of the objective function. Uses past results to intelligently decide which combination to try next. Most efficient for expensive-to-evaluate models.
Grid search — combinatorial explosionTotal trials = |H₁| × |H₂| × ... × |Hₙ|    (grows exponentially → "curse of dimensionality")
Acquisition function (Bayesian) — Expected ImprovementEI(x) = 𝔼[max(0, f(x) − f(x*))]    balance: exploit best vs explore unknown
