Linear Algebra

Linear algebra is the language ML models speak. Your data is vectors. Your model parameters are matrices. Every prediction is a chain of matrix multiplications. Once you see this, ML architecture diagrams stop being mysterious.

Vectors — Ordered Lists with Direction

A vector is an ordered list of numbers. It represents a point in space, or a direction and magnitude. In ML, almost everything gets encoded as a vector.

🎬 Real-World Example — Netflix Ratings

You rated 5 genres: Action: 5, Romance: 1, Comedy: 4, Horror: 2, SciFi: 5. That's your preference vector [5, 1, 4, 2, 5]. Netflix compares your vector to every other user's to find your “neighbours” — people with similar taste. Two vectors pointing in the same direction in this 5D space = similar taste.

import numpy as np

# Your movie preference vector
u = np.array([5, 1, 4, 2, 5])  # action, romance, comedy, horror, scifi
v = np.array([4, 2, 5, 1, 5])  # friend's ratings

# Vector magnitude (length)
print(f"Your vector length: {np.linalg.norm(u):.2f}")

# Add vectors (combine preferences)
combined = u + v
print(f"Combined: {combined}")

# Scale a vector
doubled = 2 * u
print(f"Doubled: {doubled}")

Matrices — Tables of Transformations

A matrix is a 2D grid of numbers. Multiplying a vector by a matrix transforms it — rotating, scaling, or projecting it. This is literally what every neural network layer does to your data.

🖼 Real-World Example — Image Pixels

A 28×28 greyscale image (like an MNIST digit) is a 28×28 matrix where each cell is a brightness value 0–255. A neural network flattens it into a vector of 784 numbers and multiplies it through weight matrices to produce “this is digit 7 with 96% confidence.”

Dot Product — Measuring Similarity

The dot product of two vectors is a single number telling you how similar their directions are. Large positive = aligned (similar). Near zero = unrelated. Negative = opposite.

u · v = u₁v₁ + u₂v₂ + … + uₙvₙ = ‖u‖ ‖v‖ cos(θ) cos(θ) = 1 means identical direction · cos(θ) = 0 means perpendicular · cos(θ) = −1 means opposite
🔍 Real-World Example — Search Engines

Google encodes both your search query and every web page as high-dimensional vectors. The ranking algorithm computes cosine similarity between your query vector and each page's vector. Pages closest in vector space rank higher. The same math powers Spotify recommendations and Amazon's “you might also like.”

python · dot_product.py
import numpy as np

u = np.array([5, 1, 4, 2, 5])  # your ratings
v = np.array([4, 2, 5, 1, 5])  # friend A (similar)
w = np.array([1, 5, 1, 5, 1])  # friend B (opposite tastes)

def cosine_similarity(a, b):
    return np.dot(a, b) / (np.linalg.norm(a) * np.linalg.norm(b))

print(f"You vs Friend A: {cosine_similarity(u,v):.3f}")
# 0.961 — very similar taste!
print(f"You vs Friend B: {cosine_similarity(u,w):.3f}")
# 0.461 — quite different taste

02 —

Gradient Descent

Gradient descent is how ML models learn. It's the optimization algorithm that adjusts a model's parameters to minimize mistakes. Every major model — from linear regression to GPT — trains using some variant of this idea.

The Blind Hiker Analogy

Imagine you're blindfolded in a hilly landscape and want to reach the valley. You can't see — but you can feel which way the ground slopes. Your strategy: always step in the direction that goes downhill.

The landscape
Loss Surface

A mathematical surface where height = how bad the model is (its “loss” or error on training data).

The valley
Minimum Loss

Where the model makes the fewest errors. Where we want our parameters to end up.

Your step size
Learning Rate η

Too big → overshoot the valley. Too small → takes forever. Getting this right matters enormously.

Slope direction
Gradient ∇L

The derivative of the loss. Points uphill — so we go the opposite direction to descend.

θnew = θold − η · ∇L(θ) θ = parameters · η = learning rate · ∇L = gradient of the loss function
🏠 Real-World Example — House Price Prediction

You're building a model: price = m × sqft + b. Gradient descent measures how wrong your current m and b are, then nudges them a tiny bit in the direction that reduces the error. Repeat thousands of times → your model converges on the best-fit line through your training data.

python · gradient_descent.py
import numpy as np

# House price regression from scratch with gradient descent
X = np.array([600, 800, 1000, 1200, 1500, 2000], dtype=float)
y = np.array([150,  200,  240,  280,  350,  480], dtype=float)

# Normalize inputs (crucial for gradient descent stability)
X_norm = (X - X.mean()) / X.std()

w, b = 0.0, 0.0    # start with zero weights
lr    = 0.01        # learning rate
n     = len(X)

for epoch in range(1000):
    y_pred = w * X_norm + b          # forward pass
    loss   = ((y_pred - y)**2).mean() # MSE loss

    # Compute gradients via calculus
    dw = (2/n) * (X_norm * (y_pred - y)).sum()
    db = (2/n) * (y_pred - y).sum()

    # Step downhill
    w -= lr * dw
    b -= lr * db

    if epoch % 200 == 0:
        print(f"Epoch {epoch:4d} | Loss: {loss:.2f}")

print(f"Learned: price = {w:.1f} * sqft_norm + {b:.1f}")
⚠ Common Pitfall — Learning Rate

If η is too high (e.g. >1), the model overshoots and diverges. If too small (e.g. 0.0001), training takes forever. Common starting point: try 0.001, or use Adam which auto-adjusts the rate. Use the interactive above to see both failure modes!

Bayes' Theorem — Updating Your Beliefs

The most important formula in ML. It tells you how to update a probability when you get new evidence. Every spam filter, medical diagnosis model, and language model prior uses this.

P(A|B) = P(B|A) × P(A) / P(B) Posterior = Likelihood × Prior / Evidence
🏥 Real-World Example — Medical Test

A disease affects 1% of the population. A test has 90% sensitivity (true positive rate) and a 5% false positive rate. You test positive. What's the probability you actually have the disease?

🔎 Calculation

P(+) = 0.90×0.01 + 0.05×0.99 = 0.009 + 0.0495 = 0.0585

P(Disease|+) = 0.009 / 0.0585 ≈ 15.4%

Even with a positive test, there's only a ~15% chance. Why? Because the disease is rare (1%), so most positives come from false alarms in the large healthy population. This is why doctors order confirmatory tests — and why you shouldn't panic at a single positive result.

python · bayes_theorem.py
def bayes_update(prior, sensitivity, false_positive_rate):
    """
    prior               : P(disease) before the test
    sensitivity         : P(positive | disease) = true positive rate
    false_positive_rate : P(positive | no disease)
    Returns             : P(disease | positive test)
    """
    p_positive = sensitivity * prior + false_positive_rate * (1 - prior)
    posterior  = (sensitivity * prior) / p_positive
    return posterior

# Standard scenario (rare disease)
result = bayes_update(prior=0.01, sensitivity=0.90, false_positive_rate=0.05)
print(f"Standard: {result:.1%}")          # 15.4%

# Same test during an outbreak (10% prevalence)
result_outbreak = bayes_update(prior=0.10, sensitivity=0.90, false_positive_rate=0.05)
print(f"During outbreak: {result_outbreak:.1%}")  # 66.7%!
# Lesson: the prior (base rate) changes everything.

Correlation ≠ Causation

The most important lesson in data science. Correlation means two variables move together. Causation means one actually causes the other. Confusing them has derailed drug trials, produced biased hiring tools, and led to wasteful policy decisions.

What Is Correlation?

Pearson's r ranges from −1 to +1 and measures the linear relationship between two variables.

r = Σ[(xi − x̅)(yi − y̅)] / (n ⋅ σx ⋅ σy) r = +1: perfect positive · r = 0: no linear relationship · r = −1: perfect negative

Famous Spurious Correlations

These are real correlations found in data — but absurd as causal claims. They exist by coincidence or because of a hidden third variable (a confounder).

🍦

Ice cream sales correlate with drowning rates (r ≈ 0.85). Confounder: summer heat causes both more swimming and more ice cream buying. Ban ice cream and drownings will not fall.

🏴‍☠️

Global average temperature correlates negatively with pirate count. As pirates declined 1700s→now, temperatures rose. Pure coincidence over a long shared time span.

🎬

Nicolas Cage films per year correlates with pool drownings (r ≈ 0.67). Both trended similarly over a few years by chance. One of the most viral spurious correlations on the internet.

🧠

Shoe size correlates with reading ability in children. Confounder: age. Older children have bigger feet and better reading skills. Neither causes the other.

The 4 Relationships to Know

ScenarioCorrelated?Causal?Why?
Exercise → better moodYesYesEndorphins — mechanism established
Ice cream → drowningYesNoSummer (confounder) causes both
Wealth → longevityYesPartialMechanism exists but genetics confound
Rooster crow → sunriseYesReversedLight triggers crowing, not vice versa
🤖 Real-World ML Impact — Amazon's Biased Hiring Tool

Amazon built an ML model to screen CVs, trained on 10 years of hiring data — historically male-dominated. The model learned that certain words correlated negatively with hiring outcomes. It penalised women's groups and female pronouns. The correlation existed in the data, but the cause was bias in past hiring, not candidate quality. Amazon scrapped the tool in 2018. Correlation in training data ≠ fair or causal relationship.

🔬 Establishing Causation

The gold standard is a Randomized Controlled Trial (RCT): randomly assign subjects to treatment/control. If only the treatment differs and outcomes differ → causation established.

When experiments aren't possible, use instrumental variables, difference-in-differences, or causal graphs (DAGs). This is the field of causal inference — an active ML research area.

python · correlation_analysis.py
import numpy as np

np.random.seed(42)
n = 200

# Ice cream vs drowning — confounded by temperature
temperature = np.random.normal(25, 8, n)         # the confounder
ice_cream   = 2 * temperature + np.random.randn(n) * 5
drownings   = 1.5 * temperature + np.random.randn(n) * 4

r_raw = np.corrcoef(ice_cream, drownings)[0, 1]
print(f"Ice cream vs Drownings r = {r_raw:.3f}")
# r ≈ 0.87 — strong apparent correlation!

# Control for temperature (partial correlation)
# Residualise both on temperature to remove its effect
ice_resid  = ice_cream - np.polyval(np.polyfit(temperature, ice_cream, 1), temperature)
drow_resid = drownings - np.polyval(np.polyfit(temperature, drownings, 1), temperature)

r_partial = np.corrcoef(ice_resid, drow_resid)[0, 1]
print(f"After controlling for temp: r = {r_partial:.3f}")
# r ≈ 0.01 — correlation disappears! Always look for confounders.

# Pearson r from scratch
def pearson_r(x, y):
    xc = x - x.mean(); yc = y - y.mean()
    return (xc * yc).sum() / (np.sqrt((xc**2).sum()) * np.sqrt((yc**2).sum()))

print(f"Manual r = {pearson_r(ice_cream, drownings):.3f}")
✅ Key Takeaways

1. Always ask “what causes what?” before acting on any correlation in data.

2. Look for confounders — hidden variables that explain the correlation without any causal link between the two observed variables.

3. ML models find correlations, not causes. A model predicting loan defaults might learn that zip code correlates with defaults — but that's driven by socioeconomic confounders, not the zip code itself.

4. Use correlation to generate hypotheses, then design experiments (RCTs) to confirm or deny causation.
